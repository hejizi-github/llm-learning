## 1. 一件让人惊讶的事

用 **20 行 Python**，不写任何规则，电脑能自己「学会」区分两类东西。

先看结果再问为什么——跑一下 [perceptron.ipynb](./perceptron.ipynb)，你会看到（实际运行输出，seed=42）：

```

-----|--------|-------
   1  |    1   |  50%    ← 第一轮，基本在乱猜
   5  |    3   |  55%    ← 还在摸索
   8  |    3   |  82%    ← 开始明显变好
  14  |    0   |  100%   ← 收敛！一个错误都没有

```

它**自己调整**，不需要你告诉它怎么做。14 轮迭代，权重从全零调到完全正确。

这就是感知机。1958 年的发明，机器学习史上第一个真正能「学习」的程序。

## 2. 当时的人在担心什么

1958 年，Rosenblatt 在思考一个问题：

> 人类看一眼就能认出字母「A」——不管是印刷体还是潦草的手写体。
> 计算机能做到吗？

当时的做法是**写规则**：「如果顶部有一个尖角，并且中间有一横，那就是 A。」

你可以试试：拿出纸，试着用 if-else 写出区分「猫」和「狗」的所有规则。

- 猫耳尖 → 但是折耳猫耳朵是圆的
- 狗鼻子大 → 但是吉娃娃鼻子很小
- 猫体型小 → 但是缅因猫比很多狗还大

你写了十条规则，发现每条都有反例。再写二十条，还是有漏网之鱼。

Rosenblatt 说：**不行，不能靠写规则——要让机器从例子里自己学。**

## 3. 感知机是什么：先讲类比

假设你在期末考前决定「今晚要不要复习」。你会考虑几件事：

| 考虑的事情 | 影响有多大？ |
|-----------|------------|
| 明天有没有考试 | 非常重要 |
| 今天睡了多少小时 | 比较重要 |
| 外面天气好不好 | 不太重要 |

这三件事对你的决定影响力不一样——「明天有考试」几乎直接决定你要复习，「天气好不好」基本上无所谓。

感知机做的事情**一模一样**：

- 它有几个**输入**（就像那几件事）
- 每个输入乘以一个**权重（weight）**，代表「这件事多重要」
- 把所有「重要程度 × 信息」加起来
- 超过某个**门槛**就输出 1（复习！），否则输出 0（不复习）

---

## 4. 用数字说清楚

设输入是 x₁、x₂、x₃，权重是 w₁、w₂、w₃，门槛叫 **θ**（读作 theta）：

```
加权求和 = x₁×w₁ + x₂×w₂ + x₃×w₃

如果 加权求和 > θ，输出 1
否则，输出 0
```

**用代码说就是三行：**

```python
weighted_sum = x[0]*w[0] + x[1]*w[1] + x[2]*w[2]
output = 1 if weighted_sum > threshold else 0
```

---

## 5. 它怎么「学」？

关键来了：**感知机会自己调整权重。**

初始时，权重全是随机数（乱猜）。然后：

1. 拿一个例子，算出输出
2. 如果输出**答对了** → 权重不变
3. 如果输出**答错了** → 调整权重（错哪个方向就往相反方向调一点点）

调整公式（第一次看不用记，跑完代码会自然懂）：

```python
# 答错了：
w[i] = w[i] + 学习率 × (正确答案 - 我的答案) × x[i]
```

「学习率」是一个小数（比如 0.1），控制每次调多少。

**重复这个过程几十次，权重慢慢变对，感知机就学会了。**

## 7. 感知机能做什么，做不到什么

**能做**：把两类能用一条直线分开的东西分开。

**做不到**：下面这个情况——

**XOR（异或）** 的意思是：两个输入「不一样」就输出 1，「一样」就输出 0。
（比如 0 和 1 不一样 → 输出 1；1 和 1 一样 → 输出 0。）

```
x₁  x₂  | 正确答案
 0   0  |    0    ← 一样，输出 0
 0   1  |    1    ← 不一样，输出 1
 1   0  |    1    ← 不一样，输出 1
 1   1  |    0    ← 一样，输出 0  ← 这个！
```

你画出来会发现：这四个点用**一条直线**根本没法把 0 和 1 分开。

这个问题叫 **XOR 问题**，1969 年 Minsky 和 Papert 证明了感知机永远解不了它。

→ 这直接导致了「AI 寒冬」——人们失去了对神经网络的兴趣，整整沉寂了十几年。

## 8. 历史意义

感知机是第一个证明了「**机器可以从例子中自动学习**」的系统。

Rosenblatt 当时有多兴奋？他在论文里写：

> 感知机可能是……第一个非生物系统，能以类似人类的方式感受、识别并辨别外部环境。

这是 1958 年。没有 GPU，没有 Python，计算机还是用真空管运行的。

他的这个想法，经过 60 年的演化，成为今天的神经网络、GPT、Stable Diffusion……

## 数学小补丁（可选，给想深入的同学）

> 如果你想知道「权重更新公式从哪里来」：

感知机学习规则等价于最小化分类错误。每次更新：

```
Δwᵢ = η × (y - ŷ) × xᵢ
```

其中：
- η（eta）= 学习率，控制步长
- y = 正确答案（0 或 1）
- ŷ（y-hat）= 感知机输出的答案
- xᵢ = 第 i 个输入

当 `y - ŷ = 0`（答对了），权重不动。  
当 `y - ŷ = +1`（应该输出 1 但输出了 0），权重加大。  
当 `y - ŷ = -1`（应该输出 0 但输出了 1），权重减小。

Rosenblatt 1958 年论文证明：**如果数据是线性可分的，感知机一定会收敛**（有限步内找到正确权重）。这叫「感知机收敛定理」。
