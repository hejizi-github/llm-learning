# 节点12：LLaMA 与开源爆炸（2023）

> **发展脉络位置**：InstructGPT 证明了 RLHF 能让 LLM 变得有用，但它是 OpenAI 的私有黑盒。  
> 2023 年初，Meta 的一次"学术开放"意外引爆了整个开源生态——  
> 当研究者们拿到 LLaMA 的权重，他们发现：**大模型并不是只有科技巨头才玩得起的游戏**。

---

## 一、2023年之前：LLM 的围墙花园

2022 年底，ChatGPT 横空出世，全世界都在讨论它。但如果你想自己研究它、改进它、在上面做实验——你做不到。

**你唯一的选择是通过 API 调用**：

- **GPT-3/GPT-4（OpenAI）**：需要申请访问权限，按 token 计费，模型内部对研究者不透明
- **PaLM（Google）**：完全不对外开放
- **Claude（Anthropic）**：同样是 API 黑盒

这对学术研究者来说是个巨大的障碍：

| 问题 | 具体影响 |
|------|----------|
| 无法访问权重 | 不能做微调研究、不能做解释性分析 |
| 推理成本高 | 每次实验都要花钱，贫困学生/机构无法参与 |
| 不透明 | 研究可复现性极低 |
| 算力需求 | 训练 175B 参数需要数百万美元 |

学术界急需一个**开放的、高质量的大型语言模型**。

---

## 二、LLaMA 出现：小身材，大能力

### 2.1 LLaMA-1 的核心思路

2023 年 2 月，Meta AI 的 Hugo Touvron 团队发布了 **LLaMA（Large Language Model Meta AI）**：

> 论文：Touvron et al. (2023), *LLaMA: Open and Efficient Foundation Language Models*  
> arXiv: [2302.13971](https://arxiv.org/abs/2302.13971)

LLaMA 的核心论点很反常识：

**"模型越大不一定越好——数据质量更重要。"**

之前的思路（受 Kaplan 等人 2020 Scaling Laws 影响）是：给定固定计算预算，最优策略是尽量训练大模型。但 Chinchilla（DeepMind，2022）指出，训练数据量同样重要，很多大模型其实是数据不足的。

LLaMA 更进一步：**用更多高质量数据训练相对小的模型**，让模型在推理阶段成本更低。

### 2.2 LLaMA-1 的参数规格

| 版本 | 参数量 | 训练数据 | 训练 Tokens |
|------|--------|----------|-------------|
| LLaMA-7B | 70亿 | 公开文本 | 1.0 万亿 |
| LLaMA-13B | 130亿 | 公开文本 | 1.0 万亿 |
| LLaMA-33B | 330亿 | 公开文本 | 1.4 万亿 |
| LLaMA-65B | 650亿 | 公开文本 | 1.4 万亿 |

**关键数字**：LLaMA-13B 在大部分测试上超越了 GPT-3 175B。

这意味着：不需要 1750 亿参数，130 亿就够了——前提是你用好数据充分训练。

### 2.3 训练数据构成

LLaMA 只用了**完全公开可获取的数据**：

| 数据来源 | 占比 | 说明 |
|----------|------|------|
| CommonCrawl | 67% | 网络爬取（经清洗） |
| C4 | 15% | 精心清洗的网页文本 |
| GitHub | 4.5% | 代码 |
| Wikipedia | 4.5% | 多语言 |
| Books | 4.5% | Gutenberg + Books3 |
| ArXiv | 2.5% | 学术论文 |
| Stack Exchange | 2% | Q&A |

## 三、权重泄漏：意外引爆的开源革命

### 3.1 事情的经过

LLaMA 发布时，Meta 的策略是"受限学术开放"：研究者需要**申请表格**，审核通过才能获得模型权重。这不是真正的开源——权重不能用于商业用途，也无法公开再分发。

**但是……**

- 2023 年 3 月 2 日：获得批准的研究者开始收到权重文件
- **2023 年 3 月 3 日**：有人将完整权重发布到 4chan（一个匿名论坛）
- 随后数小时：Hugging Face、各种 GitHub 仓库纷纷转载

Meta 试图删除文件，但在去中心化互联网面前，这场 Whack-a-Mole 游戏注定失败。

### 3.2 为什么这件事改变了一切

泄漏之前：研究者需要申请、等待、可能被拒绝。  
泄漏之后：任何人都可以在自己的电脑上运行一个和 GPT-3 同档次的模型。

**接下来发生了什么（仅 2023 年上半年）**：

| 时间 | 项目 | 机构 | 说明 |
|------|------|------|------|
| 2023-03-13 | Alpaca | Stanford | 52K 指令数据微调 LLaMA-7B |
| 2023-03-30 | Vicuna | UC Berkeley | GPT-4 对话数据微调 |
| 2023-04-03 | Koala | UC Berkeley | 对话模型 |
| 2023-04-12 | WizardLM | Microsoft Research | Evol-Instruct 方法 |
| 2023-04-24 | MiniGPT-4 | KAUST | 多模态对话 |

这是一个**压缩版的进化大爆发**——几周内出现了几十个新模型，每一个都基于 LLaMA 权重。

## 四、Alpaca：500美元复现 ChatGPT 能力

**Stanford Alpaca**（Taori et al., 2023）展示了一个令人震惊的结论：

> 用 **$500 的 GPT-3.5 API 费用**生成 52,000 条指令数据，然后微调 LLaMA-7B，就能得到接近 `text-davinci-003`（GPT-3.5）的对话能力。

### 方法：Self-Instruct

**Self-Instruct**（Wang et al., 2022，arXiv:[2212.10560](https://arxiv.org/abs/2212.10560)）的核心思路：

1. 从 175 条人工编写的"种子任务"出发
2. 让一个强大的 LLM（GPT-3.5）自动生成更多的`(指令, 输入, 输出)`三元组
3. 过滤掉质量差的，保留高质量的
4. 用这些数据微调一个小模型

**为什么有效**？因为"理解一个指令并遵从它"和"凭空生成好内容"是不同的能力。LLaMA-7B 已经有了足够的语言知识，只是不知道"怎么按指令行事"——少量指令数据就能让它学会这个格式。

```
一条典型的 Alpaca 训练样本：

指令（Instruction）：
"用一句话解释机器学习和深度学习的区别。"

输入（Input）：
""（本题无需额外输入）

输出（Output）：
"机器学习是让计算机从数据中学习规律的技术，
 深度学习是机器学习的一个子集，专门使用多层神经网络来学习。"
```

## 五、LoRA：让每个人都能微调大模型

### 5.1 问题：全参数微调太贵了

假设你想在自己的领域（比如医疗问答）微调 LLaMA-7B：

- **全参数微调（Full Fine-tuning）**：7B 个参数都要存储、计算梯度、更新
- 所需 GPU 显存：约 28 GB（FP16）+ 优化器状态 = **~80 GB 以上**
- 这超过了消费级 GPU（A100 80GB 是企业级，RTX 3090 只有 24GB）

**LoRA（Low-Rank Adaptation）** 解决了这个问题。

> 论文：Hu et al. (2021), *LoRA: Low-Rank Adaptation of Large Language Models*  
> arXiv: [2106.09685](https://arxiv.org/abs/2106.09685)

### 5.2 数学直觉：什么是"低秩"

**先从线性代数说起**（初中友好版）：

假设你有一个矩阵 W，形状是 1000×1000（共 100 万个数）。  
现在我想"微调"它，也就是给它加一个变化量 ΔW，也是 1000×1000 的。

**LoRA 的关键发现**：在微调过程中，ΔW 其实是"低秩"的——它看起来复杂，但本质上可以被两个小矩阵的乘积精确（或近似）表示：

```
ΔW = B × A

其中：
- B 的形状是 1000 × r（1000行，r列）
- A 的形状是 r × 1000（r行，1000列）
- r 远远小于 1000（比如 r=8）
```

**这意味着什么**？

| 方法 | 需要存储的参数量 | 举例（d=1000, r=8） |
|------|-----------------|---------------------|
| 全参数 ΔW | d × d = 1,000,000 | 100 万个数 |
| LoRA（A+B） | d×r + r×d = 2dr | 1.6 万个数（缩小 62.5 倍！） |

**为什么叫"低秩"**？

"秩（rank）"是线性代数里描述矩阵"独立信息量"的概念。一个 1000×1000 的矩阵最多有 1000 的秩；但通过 B×A 分解，我们强迫 ΔW 的秩最多只有 r。

这听起来是个限制——但实验发现，**微调时真正重要的权重变化，其实内在维度很低**。r=8 就足够了。

### 5.3 LoRA 的前向传播

训练时的完整公式：

```
h = W₀x + ΔWx = W₀x + BAx

其中：
- W₀：原始预训练权重，冻结（frozen），不计算梯度
- B：形状 d×r，初始化为全零
- A：形状 r×d，随机初始化（高斯）
- x：输入向量
- h：输出向量
```

**初始化很关键**：B 初始为零，所以训练开始时 ΔW = B×A = 0，模型输出和原始一样。这保证了训练稳定性。

推理时，可以把 ΔW 加到 W₀ 里，完全不增加推理延迟：

```
W_final = W₀ + BA（合并，变回标准矩阵）
```

### 5.4 LoRA 只改哪些层？

LoRA 通常只应用于 **Transformer 里的注意力权重矩阵**（Q、K、V、O），而不是所有权重。这进一步减少了需要训练的参数。

对于 LLaMA-7B，用 r=8 的 LoRA：
- 全参数微调：7B 参数需要更新
- LoRA 微调：约 **4-20M 参数**（少于 0.3%），显存需求降到 **12-16 GB**，RTX 3090 可以跑

### 5.5 缩放因子 α

实际使用中，LoRA 有一个额外的超参数 α，最终的更新公式是：

```
h = W₀x + (α/r) × BAx
```

当 r 变化时，用 α/r 保持更新幅度稳定。通常 α = 2r（即 α/r = 2）是一个常见默认值。

## 六、QLoRA：把量化和 LoRA 结合起来

### 6.1 问题：LoRA 已经够省了，还不够？

LoRA 减少了**可训练参数**，但原始模型权重 W₀ 仍然需要加载到显存里（FP16 下 LLaMA-65B 需要 130 GB）。

**QLoRA（Dettmers et al., 2023）** 再进一步：将 W₀ 量化为 **4-bit 精度**。

> 论文：Dettmers et al. (2023), *QLoRA: Efficient Finetuning of Quantized LLMs*  
> arXiv: [2305.14314](https://arxiv.org/abs/2305.14314)

### 6.2 量化是什么？

**量化（Quantization）**：用更少的 bit 来存储数字，牺牲一点精度换来大幅省内存。

| 精度 | 每个数用多少 bit | 7B模型显存占用 |
|------|-----------------|---------------|
| FP32 | 32 bit | ~28 GB |
| FP16/BF16 | 16 bit | ~14 GB |
| INT8 | 8 bit | ~7 GB |
| **NF4（QLoRA用）** | **4 bit** | **~3.5 GB** |

**NF4（4-bit NormalFloat）**：QLoRA 发明的特殊量化格式，针对神经网络权重的正态分布特性做了优化，比普通 INT4 精度损失更小。

**QLoRA 实际效果**：65B 参数的 LLaMA，可以在**单张 48GB A40 GPU**上微调，而以前需要多台 A100 服务器。

## 七、LLaMA-2：官方拥抱开源

2023 年 7 月，Meta 正式发布 **LLaMA-2**（Touvron et al., 2023，arXiv:[2307.09288](https://arxiv.org/abs/2307.09288)）：

| 特性 | LLaMA-1 | LLaMA-2 |
|------|---------|---------|
| 最大规模 | 65B | 70B |
| 上下文长度 | 2048 tokens | **4096 tokens** |
| 许可证 | 受限学术 | **商业友好（非封闭）** |
| 对齐方式 | 无 | **RLHF（200万人工标注）** |
| 注意力机制 | MHA | **GQA（加速推理）** |

LLaMA-2-Chat（对话版）与 ChatGPT 进行了正面比较，在很多评估上非常接近。

**重要意义**：Meta 这次正式给了商用许可（有条件），标志着 Meta 将开源战略作为长期选择——用生态与 OpenAI 竞争，而不是靠自己的封闭模型。

## 八、PEFT 生态的百花齐放

LoRA 只是 **PEFT（Parameter-Efficient Fine-Tuning，参数高效微调）** 大家族的一员。2023 年，整个 PEFT 生态蓬勃发展：

| 方法 | 原理 | 优势 |
|------|------|------|
| **LoRA** | 低秩矩阵分解 | 灵活，无推理延迟 |
| **Prefix Tuning** | 给每层输入加可训练前缀向量 | 无需修改权重 |
| **Prompt Tuning** | 只训练输入的软 prompt 向量 | 极少参数 |
| **Adapter** | 在层之间插入小网络 | 可模块化组合 |
| **IA³** | 缩放激活值 | 参数量极少 |

Hugging Face 的 `peft` 库将这些方法统一封装，几行代码即可使用任何 PEFT 方法。

## 九、数学小补丁：矩阵乘法回顾

> 适合第一次看到 BA 分解的读者

如果 B 是 m×r 的矩阵，A 是 r×n 的矩阵，那么：

```
C = B × A  →  形状是 m×n

C[i][j] = Σ(k=0 到 r-1) B[i][k] × A[k][j]
```

举个具体例子（r=2，m=3，n=3）：

```
B = [[1, 0],    A = [[1, 2, 3],
     [0, 1],         [4, 5, 6]]
     [2, 3]]

B×A = [[1*1+0*4, 1*2+0*5, 1*3+0*6],
        [0*1+1*4, 0*2+1*5, 0*3+1*6],
        [2*1+3*4, 2*2+3*5, 2*3+3*6]]
     = [[1, 2, 3],
        [4, 5, 6],
        [14, 19, 24]]
```

关键直觉：**两个小矩阵的乘积，可以表示一个大矩阵的低秩结构**。

## 十、局限与下一步

### LLaMA/LoRA 时代的局限

| 局限 | 具体表现 |
|------|----------|
| **仍然落后闭源** | LLaMA-2-70B 在复杂推理上还是输给 GPT-4 |
| **对话对齐成本** | RLHF 需要大量人工标注，小机构难以复现 |
| **长上下文能力弱** | 4096 tokens 对很多任务不够 |
| **多模态缺失** | 只有文本，GPT-4V 已经能看图 |
| **量化有精度损失** | QLoRA 的 NF4 在某些任务上有退化 |

### 催生下一个突破：更简洁的对齐方法

RLHF（InstructGPT 的对齐方法）需要训练奖励模型 + PPO，复杂且不稳定。

这引发了一个问题：**能不能绕开 RL，直接用监督学习做对齐？**

**DPO（Direct Preference Optimization，Rafailov et al., 2023）** 就是这个问题的答案。它把奖励最大化问题转化为一个直接的分类损失，去掉了奖励模型和 PPO，让对齐变得像普通的监督微调一样简单。

> **下一节点**：[节点13 DPO：更简洁的对齐（2023）](./13-dpo-2023.md) — 绕开奖励模型，直接优化人类偏好
