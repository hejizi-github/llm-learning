# 节点09：GPT-2（2019）——单向放大，文本生成能力的第一次爆发

> **核心一句话**：GPT-2 用一个简单却强力的赌注证明了"把语言模型做大，它会自动学会做很多任务"——这是通往 GPT-3 和 ChatGPT 的关键一步。

---

## 背景故事：BERT 解了理解，但生成还差得远

### BERT 的成功与缺口

2018 年末，BERT 横空出世，在 11 项 NLP 理解任务上刷新纪录。研究者们欢呼雀跃：机器终于能"读懂"上下文了。但 BERT 有一个先天缺陷：它是**双向**的编码器，每个词能同时看到左边和右边的上下文。

这对"理解"很好，对"生成"却是致命的问题——当你让模型一个词一个词地写文章时，第一个词还没写出来，右边的词根本不存在。双向注意力在生成场景下无法使用。

类比一下：BERT 像一位博览群书、理解力极强的编辑，却不会写作；GPT 系列则是一位能滔滔不绝下笔的作家，从第一个字开始就自己往后写。

### GPT-1 的局限

2018 年 6 月，OpenAI 发布 GPT-1（也叫"GPT"）。它用 Transformer Decoder 做**单向**语言模型预训练，然后在下游任务上 fine-tune。GPT-1 有 117M 参数，在 BooksCorpus 上训练——效果不错，但仍需要任务专用的 fine-tuning 数据。

GPT-1 留下了两个未解之问：

1. 如果把模型做得更大、训练数据更多，会发生什么？
2. 有没有可能完全不用 fine-tuning，直接用预训练模型完成任务？

GPT-2 就是这两个问题的答案。

---

## 原理讲解：单向因果语言模型

### 什么是"因果语言模型"

GPT-2 的核心思想非常简单：**预测下一个词**。

给定一段文字的前 $t-1$ 个词，预测第 $t$ 个词是什么。用数学写出来：

$$P(x_t \mid x_1, x_2, \ldots, x_{t-1})$$

读作："在已经看到第1到第 $t-1$ 个词的条件下，第 $t$ 个词是什么的概率。"

把整篇文章的概率写出来，就是把每一步"预测下一个词"的概率连乘：

$$P(x_1, x_2, \ldots, x_T) = \prod_{t=1}^{T} P(x_t \mid x_1, \ldots, x_{t-1})$$

这叫**自回归语言模型**（Autoregressive Language Model），因为每一步都依赖自己之前的输出往后生成。

相比之下，BERT 的 MLM（遮罩语言模型）是"从中间挖洞再填回去"，不是从左到右预测下一个词——所以 BERT 不能直接用来生成文章。

### 训练目标：交叉熵损失

模型的训练目标是让"预测正确词"的概率尽量高。对应的损失函数叫**交叉熵损失**：

$$\mathcal{L} = -\frac{1}{T} \sum_{t=1}^{T} \log P(x_t \mid x_1, \ldots, x_{t-1})$$

**初中生版解释**：

- $\log P(\cdot)$ 是"正确答案的概率取对数"。概率越高（模型越自信且正确），对数越接近 0；概率越低，对数是越大的负数。
- 前面加个负号，变成正数，我们想**最小化**它。
- 对所有位置求平均，就是"平均每个词预测错多少"。
- 直觉：损失越小 → 模型对每一个词的预测越准确。

### 架构：Transformer Decoder 的 Scale-Up

GPT-2 的网络结构和 GPT-1 本质相同——都是 **Transformer Decoder 堆叠**，每一层只能"向左看"（通过因果注意力掩码）。核心变化是**规模大幅放大**：

| 版本 | 层数 | 隐向量维度 | 注意力头数 | 参数量 |
|------|------|-----------|-----------|--------|
| GPT-1 | 12 | 768 | 12 | 117M |
| GPT-2 small | 12 | 768 | 12 | 117M |
| GPT-2 medium | 24 | 1024 | 16 | 345M |
| GPT-2 large | 36 | 1280 | 20 | 762M |
| GPT-2 XL | 48 | 1600 | 25 | 1.5B |

上下文窗口也从 GPT-1 的 512 个 token 扩展到 **1024 个 token**，意味着模型每次能"记住"的文字多了一倍。

### Pre-LN vs Post-LN：一个关键的工程细节

Transformer 原始论文（[vaswani2017]）用的是 **Post-LN**（后归一化）：先做注意力或 FFN，再做 Layer Norm。

GPT-2 改成了 **Pre-LN**（前归一化）：先做 Layer Norm，再做注意力或 FFN。

```
Post-LN（原始 Transformer）:
  x → Attention → x + residual → LayerNorm → 输出

Pre-LN（GPT-2）:
  x → LayerNorm → Attention → x + residual → 输出
```

**为什么这个改动很重要？**

- Post-LN 在深层网络训练时梯度容易不稳定，需要精心调整学习率。
- Pre-LN 把归一化放在最前面，梯度流动更稳定，允许更深的网络稳定训练。
- 这个小改动让 GPT-2 能堆到 48 层而不崩溃，是后来 GPT-3（96 层）能成功的重要基础。

## 规模化：从数据开始

### WebText：互联网的高质量子集

GPT-1 在 BooksCorpus（约 1GB 文本）上训练；GPT-2 自己构建了一个全新数据集——**WebText**。

构建方法：爬取 Reddit 上所有**被点赞超过 3 次**的外链网页，过滤掉维基百科和新闻，最终收集了约 **40GB 的文本**，覆盖 800 万个网页。

为什么用 Reddit 点赞来过滤？

> Reddit 用户会帮你做筛选：一个链接被很多人认为值得点赞，说明内容有一定质量，写作连贯，有意思。这是众包的质量过滤器。

相比 GPT-1，训练数据扩大了 40 倍以上，内容从书籍扩展到了网络上多样化的真实写作风格。

## Few-Shot Prompting 的涌现

GPT-2 论文（Radford et al., 2019，OpenAI Blog，无 arXiv 版；详见 <https://openai.com/research/better-language-models>）中最令人震惊的发现是：**一个从未专门为任何任务训练过的语言模型，仅靠提示（prompt）就能完成多种任务**。

这催生了三个重要概念：

### Zero-Shot（零样本推理）

不给任何示例，直接问。

```
提示（Prompt）：
Translate English to French: cheese =>

GPT-2 的输出：
fromage
```

模型从未被专门训练翻译任务，只是见过大量英法对照的网页文本，自然学会了翻译模式。

### One-Shot（单样本推理）

给一个示例，然后问。

```
提示：
Translate English to French:
sea otter => loutre de mer
cheese =>

GPT-2 的输出：
fromage
```

给了一个例子之后，模型更好地理解了"这是在做翻译任务"，准确率提升。

### Few-Shot（少样本推理）

给几个示例，然后问。

```
提示：
Translate English to French:
sea otter => loutre de mer
peppermint => menthe poivrée
plush giraffe => girafe peluche
cheese =>

GPT-2 的输出：
fromage
```

示例越多，模型越能准确把握任务格式和要求，输出也更稳定。

**这有什么深远意义？**

在 GPT-2 之前，让模型做一个新任务，必须收集标注数据、专门 fine-tune。GPT-2 展示了一种可能性：大模型可以直接通过"文字描述任务"来完成任务，不需要梯度更新。这个现象后来被称为**上下文学习**（In-Context Learning），是 GPT-3 的核心能力，也是 ChatGPT 能和人类对话的基础之一。

## Scale Law 的早期迹象

GPT-2 的实验结果给研究者留下了一个强烈的感觉：**模型越大，效果越好，并且这个趋势好像没有消失的迹象**。

从 117M 到 1.5B，四个版本的 GPT-2 在语言建模任务上的困惑度（perplexity，越低越好）持续下降，没有明显的"瓶颈"。

这个观察促使 OpenAI 的 Kaplan 等人进行了系统性研究。2020 年，他们发表了论文《Scaling Laws for Neural Language Models》（[kaplan2020scaling]），揭示了三条神奇的规律：

1. **模型参数量 $N$** 增大 → 损失按幂律下降：$\mathcal{L}(N) \propto N^{-\alpha_N}$
2. **训练数据量 $D$** 增大 → 损失按幂律下降：$\mathcal{L}(D) \propto D^{-\alpha_D}$
3. **计算量（FLOPs）$C$** 增大 → 损失按幂律下降：$\mathcal{L}(C) \propto C^{-\alpha_C}$

**幂律是什么意思？**

简单说：如果把数据量翻 10 倍，损失会以一个固定的比例下降；再翻 10 倍，还是以同样的比例下降——这个趋势在很大的范围内保持稳定，没有"到顶了"的迹象。

这个发现极为重要，因为它意味着：**投入更多算力和数据，效果会可预测地变好**。这成了 GPT-3、GPT-4 乃至 Claude 系列背后的底层逻辑——不是"碰运气"，而是工程上可规划的"规模红利"。

## 局限与衔接：没有 RLHF，生成内容难以控制

GPT-2 虽然会生成流畅的文字，但它并不"听话"。它会：

- 在对话中自顾自地跑题
- 编造事实（"幻觉"问题）
- 在没有任何限制的情况下生成有害内容

根本原因是：GPT-2 的训练目标是**预测下一个词**，而不是"生成对人类有帮助的内容"。它学的是"互联网上的人怎么写作"，互联网上什么都有——脏话、假新闻、阴谋论也都有。

这个问题后来通过 **RLHF**（人类反馈强化学习，Reinforcement Learning from Human Feedback）来解决：

1. 先用大量数据预训练一个语言模型（GPT-3）
2. 让人类标注员对模型输出打分，训练一个"奖励模型"
3. 用强化学习让语言模型输出让人满意的内容

OpenAI 的 **InstructGPT**（2022）和后来的 **ChatGPT** 正是沿着这条路走来的。

---

## 局限催生下一代突破

| GPT-2 的问题 | 解决方案 |
|------------|---------|
| 1.5B 参数已是上限，zero-shot 能力不稳定 | **GPT-3**（2020）：1750 亿参数，few-shot 能力质的飞跃 [[kaplan2020scaling]] |
| 生成内容难以控制，容易产生有害内容 | **InstructGPT**（2022）：RLHF 对齐人类意图 |
| 没有对话格式，无法自然问答 | **ChatGPT**（2022）：多轮对话 + RLHF |
| 上下文窗口只有 1024 token | **GPT-3** 扩展到 2048；**GPT-4** 扩展到 128K |
| 只会生成文字，无法理解图像 | **GPT-4V**（2023）：多模态 |
| 没有 fine-tune，下游任务准确率受限 | **GPT-3.5 / GPT-4**：指令 fine-tune（SFT）|
