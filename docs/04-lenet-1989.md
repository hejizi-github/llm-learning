# 节点 04：LeNet — 让网络学会"看"（1989–1998）

> 前置知识：[节点 03 — 反向传播（1986）](03-backprop-1986.md)  
> 引用来源：[@lecun1989zipcode], [@lecun1998lenet]

## 故事的起点：一个邮局里的问题

1989 年，美国贝尔实验室（Bell Labs）。  
Yann LeCun 面对的问题很具体：**美国邮政每天要处理数亿封信，手写邮政编码必须人工看——成本太高**。

能不能让机器自动识别手写数字？

当时已经有了反向传播（节点 03），所以理论上可以用神经网络来训练一个分类器。  
LeCun 试了——**效果很差**。

为什么？这就是本节的核心问题。

---

## 旧方法的死穴：全连接网络看图片

回忆反向传播那节的网络：每个输入连到每个隐藏节点，每个隐藏节点连到输出。  
这叫**全连接**（fully connected）。

把一张 28×28 像素的手写数字图片，拉成一条长度 784 的向量，输入全连接网络。

问题立刻出现：

### 问题 1：参数太多了

假设隐藏层有 1000 个节点：  
- 输入 → 隐藏：`784 × 1000 = 784,000` 个权重  
- 训练数据才几千张图，参数比数据多 100 倍，**严重过拟合**。

### 问题 2：不懂"空间结构"

全连接把图片拉成了向量，完全**丢掉了像素的位置关系**。  
"这个像素在左上角"和"这个像素在右下角"对全连接来说没有区别。  
但人眼识别数字时，**相邻像素之间的关系**才是关键——"这里有一条竖线，旁边有一条横线"。

### 问题 3：不懂"平移不变性"

把一个"7"向左移 3 个像素，全连接就看不出来了——  
因为同样的竖线在输入向量里跑到了完全不同的位置。

LeCun 的答案，是重新设计网络的结构。

---

## 核心思想 1：卷积——用一个小"探测器"扫描图片

### 直觉

想象你拿着一个放大镜，上面刻着一个小图案（比如一条斜线）。  
你拿着这个放大镜，在整张图上**从左到右、从上到下**扫描，每扫一个位置，记录"这里和我的图案有多像"。

扫完一遍，你就得到了一张新图——上面每个位置的数字，表示"原图在这里有多像斜线"。  
这个过程就叫**卷积**（convolution）。

那个放大镜上的图案（一组小数字），叫做**卷积核**（kernel）或**滤波器**（filter）。

### 数学（初中版）

卷积核是一个小矩阵，比如 3×3：

```
kernel = [[ 1,  0, -1],
          [ 2,  0, -2],
          [ 1,  0, -1]]
```

在图片的每个 3×3 区域，做**对应位置相乘再求和**：

```
输出 = ∑（图片区域中每个像素 × kernel中对应的数）
```

举个具体例子，图片某个 3×3 区域是：

```
图片区域    kernel      相乘
[100, 50,  0]   [ 1, 0,-1]   100*1 + 50*0 + 0*(-1)
[ 80, 40, 10]   [ 2, 0,-2] + 80*2 + 40*0 + 10*(-2)
[ 60, 30, 20]   [ 1, 0,-1] + 60*1 + 30*0 + 20*(-1)
                              = 100 + 0 + 0 + 160 + 0 - 20 + 60 + 0 - 20
                              = 280
```

这个 kernel 是一个**竖直边缘探测器**——左边明亮、右边暗淡的区域，输出值很大。

### 为什么这解决了问题？

- **参数少**：一个 3×3 kernel 只有 9 个参数，却能扫描整张图片
- **懂空间结构**：每次只看一个局部区域（**局部感受野**）
- **平移不变性**：数字向左移了，kernel 只是在不同位置扫到它，输出值一样

## 核心思想 2：池化——缩小图片，保留重要信息

卷积之后，图片还是很大。怎么让信息更紧凑？

**最大池化**（max pooling）：把图片切成 2×2 的小格，每格只保留最大值。

```
原图（4×4）：          池化后（2×2）：
1  3  2  4            3  4
5  6  7  8    →       6  8
9  2  1  3            9  5
0  4  5  6            6  7
```

（每个 2×2 格取最大值）

效果：
- 图片缩小一半（宽和高各缩半），参数少了 4 倍
- "这里有一条竖线"的信息被保留了，"竖线在格子里的精确位置"被丢掉了
- 对小幅平移更鲁棒

## LeNet 的整体结构

LeCun 1989 年发表的网络（简化版，全称见 1998 LeNet-5）：

```
输入图片 (28×28)
      ↓
  卷积层 1 (6个 5×5 kernel) → 24×24×6 的特征图
      ↓
  池化层 1 (2×2 max) → 12×12×6
      ↓
  卷积层 2 (16个 5×5 kernel) → 8×8×16
      ↓
  池化层 2 (2×2 max) → 4×4×16
      ↓
  拉平（flatten） → 256 个数
      ↓
  全连接层 → 120 → 84 → 10
      ↓
  softmax → 10类数字的概率
```

**参数数量**：约 6 万个（和 784×1000 = 78 万相比，小了 13 倍！）

---

## 数学补丁：什么是"特征图"

卷积层 1 用了 6 个不同的 kernel，每个 kernel 扫一遍图，得到一张 24×24 的图。  
6 个 kernel → 6 张图，叠在一起就是 24×24×6 的**特征图**（feature maps）。

每个 kernel 学到不同的"探测技能"：
- kernel 1 可能学到"竖线探测"
- kernel 2 可能学到"横线探测"  
- kernel 3 可能学到"斜线探测"
- ……（这些不是人工指定的，是反向传播学出来的）

---

## 这一切怎么训练？

还是反向传播！只是现在需要计算卷积层的梯度。

好消息：卷积本质上是一个线性操作（乘法 + 加法），  
反向传播的链式法则完全适用。

训练步骤：
1. 前向传播：图片 → 卷积 → 池化 → 全连接 → 预测
2. 计算损失（和节点 03 一样）
3. 反向传播：从输出层梯度，一层层算回来，包括卷积层的 kernel 梯度
4. 更新 kernel 权重（和普通权重一样）

## 当时的局限

1989 年的 LeNet 已经很厉害了，但：

- **只能识别数字**：训练数据是邮政编码，任务很窄
- **计算量大**：当时没有 GPU，训练一个网络要几天
- **需要精心设计**：网络结构、kernel 大小还是人工选的
- **与 SVM 竞争失败**（1990s）：当 SVM 出现时，很多任务上 SVM 比 LeNet 更好用、更可解释——CNN 沉寂了近 20 年

---

## 历史意义

1998 年的 LeNet-5 论文，建立了现代 CNN 的基本结构：
- **卷积层 → 池化层 → 全连接层**的堆叠
- 用反向传播端到端训练，不需要手工设计特征
- 引入 MNIST 数据集（60,000 张手写数字），成为之后 20 年的标准 benchmark

直到 2012 年，AlexNet（节点 05，敬请期待）在更大的数据集上证明了 CNN 的潜力，深度学习才真正爆发。
