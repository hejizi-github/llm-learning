## 时代背景：DDPM 的一个致命缺陷

2020 年，DDPM 在图像质量上超越了 GAN。但它有一个让工程师头疼的问题：

**太慢了。**

想象你用 DDPM 生成一张 256×256 的图片：
- 需要做 **1000 步**反向去噪
- 每一步都要调用一次神经网络
- 在普通 GPU 上，这要等 **好几分钟**

相比之下，GAN 只需要一次前向传播，几毫秒就能出图。

2020 年，宋飏（Jiaming Song，Stanford）等人发表了 DDIM 论文，用一个优雅的思路把采样步数从 1000 步压缩到了 **10-50 步**，速度提升了 **10-100 倍**，图像质量却几乎没有下降。

> 📄 原论文：Song, J., Meng, C., & Ermon, S. (2021). *Denoising Diffusion Implicit Models*. ICLR 2021. arXiv:2010.02502 [@song2020ddim]

---

## 关键洞察：DDPM 为什么必须走 1000 步？

先回忆 DDPM 的反向采样步骤（见节点16）：

```
x_{t-1} = μ_θ(x_t, t) + σ_t * z    其中 z ~ N(0, I)
```

每一步都要加一个随机噪声 `σ_t * z`。这个随机项是问题的根源：

**如果每步都加随机噪声，你就必须走很多小步，否则累积误差会让生成结果崩溃。**

这就像你蒙着眼走路——每步都可能走歪，所以要走小步。

DDIM 的思路：**去掉随机噪声，改用确定性步骤**。没有随机性 → 可以大步走 → 步数从 1000 降到 50。

---

## DDIM 的确定性采样公式

DDIM 把反向步骤改写为：

```
x_{t-1} = √(ᾱ_{t-1}) * x̂_0(x_t) + √(1 - ᾱ_{t-1}) * ε_θ(x_t, t)
```

**逐项解释**（面向初中生）：

| 符号 | 含义 | 类比 |
|------|------|------|
| `ᾱ_t` | 从第0步到第t步，保留的"原始信号"比例 | 拍照时保留的清晰度 |
| `x̂_0(x_t)` | 根据当前含噪图像预测的原始图像 | "我觉得原来的照片长这样" |
| `ε_θ(x_t, t)` | 神经网络预测的噪声 | "我觉得加进去的噪声是这些" |
| `√(1 - ᾱ_{t-1})` | 这一步保留的噪声比例 | 残余的模糊程度 |

**关键区别**：这个公式里没有随机变量 `z`！给定同样的 `x_t`，每次计算结果完全相同。

---

## 预测原始图像 x̂_0

神经网络 `ε_θ` 预测的是噪声（DDPM 的训练方式）。我们用这个预测噪声反推出原始图像估计：

```
x̂_0(x_t) = (x_t - √(1 - ᾱ_t) * ε_θ(x_t, t)) / √ᾱ_t
```

**手算验证**（用简单数字）：

假设 `ᾱ_t = 0.04`（第999步，几乎全是噪声），`x_t = 0.3`，预测噪声 `ε_θ = 0.2`：

```
x̂_0 = (0.3 - √(1 - 0.04) * 0.2) / √0.04
     = (0.3 - √0.96 * 0.2) / 0.2
     = (0.3 - 0.98 * 0.2) / 0.2
     = (0.3 - 0.196) / 0.2
     = 0.104 / 0.2
     = 0.52
```

这就是在"几乎全是噪声"的图像里，模型对原始图像数值的估计。

## 跳步：DDIM 真正的魔法

DDIM 最聪明的地方在于：**你不必每步都走**。

正常 DDPM 路径：`x_1000 → x_999 → x_998 → ... → x_1 → x_0`（1000步）

DDIM 子序列路径（步数=10）：`x_1000 → x_900 → x_800 → x_700 → ... → x_100 → x_0`（10步）

每次只需要知道起点 `ᾱ_s`（当前步）和终点 `ᾱ_t`（目标步），就可以直接"跳"过去。

**为什么可以跳步？**

因为 DDPM 的加噪公式（见节点16）允许我们直接从 `x_0` 跳到任意步 `x_t`：

```
x_t = √ᾱ_t * x_0 + √(1 - ᾱ_t) * ε
```

既然正向可以跳，反向（有了 `x̂_0` 的估计之后）也可以跳。这就是 DDIM 的核心逻辑。

## 三种采样模式对比

DDIM 实际上是一个**连续体**，通过参数 `η`（eta）控制随机程度：

| 模式 | η 值 | 特点 |
|------|------|------|
| **DDPM**（原始） | η = 1 | 每步完全随机，路径不可复现 |
| **DDIM-随机** | 0 < η < 1 | 部分随机，介于两者之间 |
| **DDIM-确定** | η = 0 | 完全确定，同起点 = 同结果 |

`η = 0` 时的完整公式：

```
x_{t-1} = √ᾱ_{t-1} * x̂_0 + √(1 - ᾱ_{t-1}) * ε_θ(x_t, t)
```

加入 η 后：

```
σ_t = η * √((1 - ᾱ_{t-1}) / (1 - ᾱ_t)) * √(1 - ᾱ_t / ᾱ_{t-1})

x_{t-1} = √ᾱ_{t-1} * x̂_0 
         + √(1 - ᾱ_{t-1} - σ_t²) * ε_θ(x_t, t)
         + σ_t * z        （z 是随机噪声）
```

η = 0 时 σ_t = 0，随机项消失。η = 1 时退化回 DDPM。

## 速度对比：数字说话

在标准 DDPM 实验（CIFAR-10，32×32图像）上：

| 采样步数 | FID 分数（越低越好） | 相对质量 |
|---------|---------------------|---------|
| 1000步（DDPM） | 3.17 | 基准 |
| 100步（DDIM） | 4.16 | 略降 |
| 50步（DDIM） | 4.67 | 可接受 |
| 10步（DDIM） | 8.31 | 明显下降 |

**实际选择**：生产环境通常用 **50步 DDIM**，可以在质量和速度之间取得最好的平衡。

> 注：FID（Frechet Inception Distance）是衡量生成图像质量的指标，越低代表生成图像与真实图像分布越接近。

## 数学小补丁：什么是"确定性"？

**随机**：相同输入 → 不同输出

```python
import random
random.seed(None)  # 不固定种子
print(random.random())  # 每次不同
```

**确定性**：相同输入 → 相同输出

```python
import random
random.seed(42)  # 固定种子
print(random.random())  # 每次都是 0.6394267984578837
```

DDPM 的采样像第一种——你永远不知道会生成什么。DDIM（η=0）像第二种——输入相同的纯噪声，每次生成同一张图。

这个特性叫做**确定性映射**：纯噪声空间 → 图像空间有一个固定的对应关系。

## 历史意义：Stable Diffusion 的核心组件

2022 年的 Stable Diffusion（节点18）正是用 DDIM 作为默认采样器。没有 DDIM，就没有那个让"文字生图"变得实用的速度。

DDIM 的贡献可以概括为：**它把扩散模型从"学术有趣"变成了"工程可用"**。

整个扩散模型生态的加速采样研究（PNDM、DPM-Solver、LCM等），都可以看作是在 DDIM 思路上的进一步发展。

## 本节小结

| 概念 | DDPM | DDIM |
|------|------|------|
| 采样方式 | 随机（马尔可夫链） | 确定性（或可调节） |
| 最少步数 | ~1000 步 | ~10 步 |
| 同起点重复 | 结果不同 | 结果相同 |
| 速度 | 慢（分钟级） | 快（秒级） |
| 图像质量（50步） | 不适用 | 仅略低于1000步DDPM |

**一句话总结**：DDIM 去掉了 DDPM 反向过程里的随机性，因此可以大步跳跃，把生成速度提升了10-100倍，让扩散模型进入了实用阶段。
