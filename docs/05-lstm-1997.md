## 背景故事：记忆，是 AI 最难解决的问题之一

1989 年，LeCun 用卷积神经网络让机器"看懂"了图片。  
但图片是静态的——每张图片都是独立的，网络看完这张图，不需要记得上张图是什么。

现实世界里有很多不是静态的任务：

- 理解一句话，你需要记得这句话**前面**说了什么
- 预测股价，你需要知道**过去几天**的走势
- 翻译语言，你需要追踪整个句子的**上下文**

这类任务叫做**序列任务**——输入是有顺序的，当前的输出依赖于之前的输入。

### 简单 RNN：第一次尝试

1980 年代，研究者提出了**循环神经网络（Recurrent Neural Network，RNN）**。  
思路很简单：给网络加一根"自反馈"的线，让它能把上一时刻的状态传给下一时刻。

```
输入 x(t) ──→ [神经元] ──→ 输出 y(t)
                  ↑  ↓
                 隐状态 h(t-1)
```

用公式表示：

$$h(t) = \tanh(W_h \cdot h(t-1) + W_x \cdot x(t) + b)$$

看起来很优雅——网络有了"记忆"（隐状态 $h$），能看到历史信息。

但问题在 1991 年被一位年轻的德国研究生发现了。

---

## 关键危机：梯度消失问题

Sepp Hochreiter 当时还是个学生，在导师 Jürgen Schmidhuber 的指导下研究 RNN。  
他发现了一个让人沮丧的事实：

> **RNN 在训练时，几乎无法学习"距离较远的依赖关系"。**

比如，有这样一个句子：

> "我昨天去了图书馆，读了三本书，最后在那里睡着了。总的来说，**这趟旅途**很有意思。"

"这趟旅途"指的是去图书馆这件事，但这个引用距离"图书馆"已经隔了很多词。  
简单 RNN 会把这个依赖关系忘掉。

**为什么会忘？因为梯度消失。**

### 数学小补丁：什么是梯度消失？

> ⚠️ **这一节涉及微积分概念，但我们会用类比来解释。**

训练神经网络的核心是**反向传播**：我们计算"损失对权重的导数"，然后更新权重。

对于 RNN，要把误差从时刻 $t=100$ 传回到 $t=1$，需要连续乘以很多个"局部梯度"：

$$\text{梯度} \propto \frac{\partial h(100)}{\partial h(1)} = \prod_{t=1}^{99} \frac{\partial h(t+1)}{\partial h(t)}$$

每个 $\frac{\partial h(t+1)}{\partial h(t)}$ 大约等于 $W_h \cdot \tanh'(\cdot)$。  
而 $\tanh$ 函数的导数最大只有 1，通常小于 1。

**类比：** 想象你在一张纸上写下一个数字 0.9，然后不断把它乘以 0.9：  
$$0.9^{10} \approx 0.35, \quad 0.9^{50} \approx 0.005, \quad 0.9^{100} \approx 0.000026$$

乘了 100 次，信号几乎变成了 0。梯度到达早期时刻时，小到可以忽略不计——权重根本学不到更新。

Hochreiter 在 1991 年的硕士论文中系统分析了这个问题 [Hochreiter, 1991]。  
Bengio、Simard 和 Frasconi 在 1994 年进一步用实验证明：随机初始化的 RNN **本质上**难以学习长程依赖 [Bengio et al., 1994]。

## LSTM 的解决方案：记忆用"线性通道"保存

Hochreiter 和 Schmidhuber 在 1997 年提出了一个绕开梯度消失的方案：**LSTM（Long Short-Term Memory，长短期记忆网络）** [Hochreiter & Schmidhuber, 1997]。

### 核心思想：细胞状态（Cell State）

LSTM 引入了一个新东西：**细胞状态 $c(t)$**，作为专门的"长期记忆"。

关键设计：细胞状态的更新尽量用**加法**，而不是乘法。

回想一下，梯度消失的根源是"连续相乘小于 1 的数"。  
但加法不会消失：$c + \text{新信息}$ 的梯度可以直接传回去，不会被压缩。

这就像一条高速公路：信息可以无损地传递很长距离。

### 三个门：精细控制记忆

LSTM 用三个"开关"（叫**门，Gate**）来控制信息的流动：

| 门名称 | 作用 | 类比 |
|--------|------|------|
| **遗忘门（Forget Gate）** | 决定细胞状态里哪些信息可以丢掉 | 大脑决定"忘掉不重要的事" |
| **输入门（Input Gate）** | 决定哪些新信息写入细胞状态 | 大脑决定"记住新看到的事" |
| **输出门（Output Gate）** | 决定从细胞状态读出哪些信息作为输出 | 大脑决定"现在说什么" |

每个门都是一个 0 到 1 之间的数，乘以信息后实现"部分通过"：  
- 门值 ≈ 1：信息完全通过（"记住"）  
- 门值 ≈ 0：信息被阻断（"忘掉"）

### 数学小补丁：sigmoid 函数

门的值用 **sigmoid 函数** $\sigma(x)$ 来产生：

$$\sigma(x) = \frac{1}{1 + e^{-x}}$$

它的输出永远在 (0, 1) 之间——正好适合做"多少信息通过"的控制。

- $x$ 很大（比如 +10）：$\sigma(x) \approx 1$，全部通过
- $x$ 很小（比如 -10）：$\sigma(x) \approx 0$，全部阻断
- $x = 0$：$\sigma(x) = 0.5$，通过一半

### LSTM 完整公式

设当前输入为 $x(t)$，上一步的隐状态为 $h(t-1)$，上一步的细胞状态为 $c(t-1)$：

**第一步：遗忘门**（决定保留多少旧记忆）

$$f(t) = \sigma(W_f \cdot [h(t-1),\ x(t)] + b_f)$$

**第二步：输入门**（决定写入多少新信息）

$$i(t) = \sigma(W_i \cdot [h(t-1),\ x(t)] + b_i)$$

$$\tilde{c}(t) = \tanh(W_c \cdot [h(t-1),\ x(t)] + b_c)$$

**第三步：更新细胞状态**（旧记忆 × 遗忘 + 新信息 × 输入）

$$c(t) = f(t) \odot c(t-1) + i(t) \odot \tilde{c}(t)$$

**第四步：输出门**（决定输出什么）

$$o(t) = \sigma(W_o \cdot [h(t-1),\ x(t)] + b_o)$$

$$h(t) = o(t) \odot \tanh(c(t))$$

其中 $\odot$ 表示逐元素相乘，$[h(t-1), x(t)]$ 表示把两个向量拼接在一起。

### 为什么这能解决梯度消失？

细胞状态的更新方程是：

$$c(t) = f(t) \odot c(t-1) + i(t) \odot \tilde{c}(t)$$

**这是加法，不是连续相乘！**  
当计算 $c(t)$ 对 $c(t-1)$ 的偏导数时，结果就是遗忘门 $f(t)$——一个可以接近 1 的值。  
只要遗忘门保持开启，梯度就能无损地传递回去，不会指数级消失。

## 代码直觉：手撕 LSTMCell

```python
import numpy as np

def sigmoid(x):
    return 1 / (1 + np.exp(-x))

class LSTMCell:
    def __init__(self, input_size, hidden_size):
        # 四组权重：f, i, g(候选), o 的权重矩阵
        scale = 0.1
        self.Wf = np.random.randn(hidden_size, input_size + hidden_size) * scale
        self.Wi = np.random.randn(hidden_size, input_size + hidden_size) * scale
        self.Wg = np.random.randn(hidden_size, input_size + hidden_size) * scale
        self.Wo = np.random.randn(hidden_size, input_size + hidden_size) * scale
        self.bf = np.zeros(hidden_size)
        self.bi = np.zeros(hidden_size)
        self.bg = np.zeros(hidden_size)
        self.bo = np.zeros(hidden_size)

    def forward(self, x, h_prev, c_prev):
        # 拼接输入和上一步隐状态
        combined = np.concatenate([h_prev, x])

        f = sigmoid(self.Wf @ combined + self.bf)   # 遗忘门
        i = sigmoid(self.Wi @ combined + self.bi)   # 输入门
        g = np.tanh(self.Wg @ combined + self.bg)   # 候选细胞状态
        o = sigmoid(self.Wo @ combined + self.bo)   # 输出门

        c = f * c_prev + i * g   # 更新细胞状态（加法！）
        h = o * np.tanh(c)       # 输出隐状态

        return h, c
```

## 局限与衔接：LSTM 之后

LSTM 是一个巨大的突破，此后十几年它一直是处理序列的主流方法。  
语音识别（Google Voice）、机器翻译的早期版本，都依赖 LSTM。

但 LSTM 也有它的局限：

### 1. 仍然串行，无法并行

LSTM 必须一步一步地处理序列——时刻 $t$ 必须等 $t-1$ 算完才能开始。  
这让 GPU 的并行能力大打折扣，训练长序列非常慢。

### 2. 注意力不够精准

LSTM 的"记忆"是一个固定大小的向量，所有过去信息都压缩进去了。  
对于非常长的序列，早期信息还是容易被遗忘或混淆。

### 3. GRU 的简化

2014 年，Cho 等人提出了 **GRU（Gated Recurrent Unit）**，把三个门简化成两个门，在很多任务上效果相当但参数更少。

### 4. Attention 和 Transformer 的崛起

2015 年，Bahdanau 等人提出**注意力机制（Attention）**：  
不只是靠压缩向量来记忆，而是让模型每一步都能"看回去"选择最相关的历史信息。

2017 年，Google 的 Vaswani 等人把"注意力"推到极致——**Transformer** 完全抛弃了循环，只用注意力机制。它可以并行训练，能处理更长的依赖，成了 GPT、BERT 的基础。

> LSTM 种下了"记忆"的种子，Transformer 长成了参天大树。
