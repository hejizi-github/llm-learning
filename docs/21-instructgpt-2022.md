# 节点21：InstructGPT — 让语言模型听人话（2022）

> **历史位置**：GPT-3（2020）→ **InstructGPT（2022-01）** → ChatGPT（2022-11）→ GPT-4（2023-03）
>
> InstructGPT 是 ChatGPT 的直接前身。它没有发明新架构，而是用"人类反馈"来教会 GPT-3 遵循指令——这个方法后来被称为 **RLHF**（Reinforcement Learning from Human Feedback，人类反馈强化学习）。

---

## 0. 一个有趣的问题：聪明≠听话

GPT-3 非常聪明——给它一段文字，它能续写得滴水不漏。但如果你问它：

> "帮我写一封道歉信"

它可能会续写出：
> "道歉信模板一：……模板二：……道歉信在人际关系中的重要性……"

因为 GPT-3 的训练目标是**预测下一个词**，它学到的是"互联网上这类提示之后通常接什么"，而不是"人类真正想要什么"。

这就是**对齐问题**（Alignment Problem）的一个具体例子：
- 模型的能力（预测下一词）
- 和人类的意图（执行指令、提供帮助）

这两件事并不天然一致。

---

## 1. 为什么预测下一词不够用？

### 1.1 三类失败模式

OpenAI 把 GPT-3 的失败模式总结为三类：

| 失败类型 | 举例 |
|---------|------|
| **不遵循指令** | 问"总结这篇文章"→ 开始分析文章结构 |
| **产生有害内容** | 问"如何…"→ 详细说明危险方法 |
| **编造事实** | 自信地说出错误的"历史事件" |

这三类问题在大量互联网文字上训练的模型里都会出现，因为互联网本身就充满了不遵循指令的内容、有害讨论和错误事实。

### 1.2 核心洞察

**不能只靠更多数据或更大模型来解决对齐问题。**

GPT-3 从 1750 亿参数扩大到更大模型，以上三类问题只会等比例放大，不会自然消失。

---

## 2. RLHF：用人类偏好来训练

InstructGPT 的核心贡献是 RLHF 三步骤流程：

```
步骤1: SFT（监督微调）
步骤2: RM（奖励模型训练）
步骤3: PPO（强化学习优化）
```

把这个类比成训练一只小狗：
- **SFT** = 先给小狗示范正确动作（"坐下"→做坐下姿势）
- **RM** = 教训狗训练员怎么判断小狗做得好不好
- **PPO** = 让小狗根据训练员的评分不断改进

---

## 3. 步骤一：SFT（监督微调）

### 3.1 什么是 SFT

SFT = Supervised Fine-Tuning，就是普通的监督学习微调。

OpenAI 雇佣了 40 名标注员，让他们：
1. 收到一个用户提示（比如"帮我解释量子纠缠"）
2. 亲手写出一个高质量的回答
3. 用这些（提示, 标注员回答）对来微调 GPT-3

**结果**：得到一个更听话的基础模型，称为 SFT 模型。

### 3.2 SFT 的局限

SFT 很有效，但有个问题：**标注员不能穷举所有可能的提示**。

如果测试时出现标注员没见过的提示类型，模型就不知道该怎么应对。这就需要第二步。

---

## 4. 步骤二：RM（奖励模型）

### 4.1 偏好数据的收集

与其让标注员写出完整回答，不如让他们做**排序**：

1. 给模型一个提示
2. 模型生成 4-9 个不同的回答
3. 标注员把这些回答从最好到最差排序

比如，对于提示"解释机器学习"，标注员可能给出排序：
```
回答C > 回答A > 回答D > 回答B
```

这样一个排序可以展开成多个**偏好对**（preference pairs）：
- (C, A)：C 好于 A
- (C, D)：C 好于 D
- (C, B)：C 好于 B
- (A, D)：A 好于 D
- (A, B)：A 好于 B
- (D, B)：D 好于 B

4 个回答的排序 → 6 对偏好数据，效率大幅提升。

### 4.2 奖励模型的结构

奖励模型（Reward Model，RM）是一个神经网络，它接受（提示, 回答）对，输出一个**分数**：

```
输入：(提示, 回答)
输出：一个数字（分数越高 = 这个回答越好）
```

训练目标：对于每个偏好对 (chosen, rejected)，让 RM 给 chosen 的分数更高。

### 4.3 Bradley-Terry 模型

RM 的训练使用 **Bradley-Terry 偏好模型**。这个模型说：

> 如果回答 A 的分数是 r_A，回答 B 的分数是 r_B，那么人类偏好 A 而不是 B 的概率是：
>
> P(A 胜于 B) = σ(r_A - r_B)

其中 σ 是 Sigmoid 函数（输出 0~1 之间的概率）。

**直觉**：分数差越大，被选中的概率越确定；分数相等时，胜率各 50%。

训练损失函数：
```
L_RM = -log σ(r_chosen - r_rejected)
```

最小化这个损失 = 让"好回答分数高于坏回答"越来越确定。

## 5. 步骤三：PPO（强化学习优化）

### 5.1 直觉：让语言模型越来越高分

现在我们有了一个"裁判"（RM），可以对任何回答打分。下一步是用这个裁判来指导语言模型改进。

这是一个**强化学习**问题：
- **智能体**：语言模型
- **动作**：生成下一个词
- **奖励**：RM 给完整回答的打分

### 5.2 KL 散度约束——别跑太远

如果只是最大化 RM 的分数，会出现一个问题：**奖励 Hacking**。

模型可能学会写出一些奇怪的、让 RM 高分但人类根本看不懂的回答。就像学生学会了"卖萌"让老师给高分，但实际没有真正学到东西。

解决方法：在奖励函数里加一个**惩罚项**，让模型不要偏离原来的 GPT-3 太远：

```
总奖励 = RM分数 - β × KL散度(新模型, 原GPT-3)
```

**KL 散度**衡量两个概率分布的"距离"——这里就是新模型和原模型在输出概率上的差异。

β 是一个可以调节的超参数，控制"多大程度上保持原来的能力"。

### 5.3 PPO 算法

PPO（Proximal Policy Optimization，近端策略优化）是一种常用的强化学习算法。关键思想：

> **每次更新不要走太大步**

类比：你在迷宫里找出口，每次只向前试探一小步，如果走错了就退回来，慢慢摸索正确路径。如果每次迈一大步，很容易走进死胡同出不来。

## 6. 数字成绩

InstructGPT（1.3B 参数）vs GPT-3（175B 参数）的人类评估：

| 指标 | 结果 |
|------|------|
| 人类偏好 InstructGPT | **85%** |
| 在 TruthfulQA 上更真实 | **+20%** |
| 有害输出减少 | **显著降低** |

**关键发现**：InstructGPT 只有 GPT-3 参数量的 1/100，但人类更喜欢它的回答。这说明**训练方式比参数规模更重要**。

## 7. 局限性

### 7.1 奖励模型欺骗（Reward Hacking）

模型可能学会"讨好"奖励模型，而不是真正变得有用。例如：写特别长的回答，因为长答案可能让 RM 评分更高；使用特定的措辞触发 RM 的高分机制。

### 7.2 标注员偏见

奖励模型的质量取决于标注员的判断。不同文化背景的标注员可能有不同的偏好，这些偏见会被编码进模型。

### 7.3 训练稳定性

PPO 在强化学习中以难以调参著称。KL 惩罚系数 β 选错了，要么模型跑偏，要么根本学不到什么。

## 9. 数学小补丁

### 9.1 对数概率（Log Probability）

语言模型输出的是**下一个词的概率分布**。比如：
```
P("今天" | "明天") = 0.3
P("明天" | "今天") = 0.4
...
```

为了数学上方便处理，我们通常取对数：
```
log P("今天" | "明天") = log(0.3) ≈ -1.2
```

对数把乘法变成加法：多个词的联合概率 = 各词对数概率之和。

### 9.2 KL 散度直觉

KL 散度（Kullback-Leibler Divergence）衡量两个概率分布的"差异"：

```python
KL(P || Q) = sum(P[i] * log(P[i] / Q[i]) for i in all_words)
```

- KL = 0：两个分布完全一样
- KL = 大数：两个分布差异很大

**类比**：两个人谈话风格的"差距"——一个人喜欢用长句，另一个喜欢用短句，KL 散度就衡量这种说话习惯的差异有多大。

### 9.3 Sigmoid 函数

Sigmoid 函数把任意实数映射到 (0, 1) 区间：

```python
def sigmoid(x):
    return 1 / (1 + exp(-x))
```

- sigmoid(0) = 0.5（50% 概率）
- sigmoid(大正数) ≈ 1（接近 100%）
- sigmoid(大负数) ≈ 0（接近 0%）

在 Bradley-Terry 模型里，sigmoid(r_A - r_B) 就是"A 好于 B 的概率"。
