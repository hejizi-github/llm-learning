# 节点22：LoRA — 大模型的"轻量级改装"（2021）

> **历史位置**：GPT-3（2020）→ InstructGPT（2022）→ **LoRA（2021/2022）** → LLaMA微调生态（2023）
>
> LoRA 解决了一个实际问题：GPT-3 有 1750 亿个参数，每次微调都要更新全部参数，需要几十张 A100 GPU。LoRA 让普通人也能微调大模型——只用 4 块消费级显卡，甚至 1 块。

---

## 0. 问题背景：微调 GPT-3 有多贵？

想象你买了一本百科全书，里面有 1750 亿个知识点。现在你想让它专门回答医学问题。

**全量微调**的做法：把整本百科全书重新印一遍，更新里面每一个知识点。
- 存储：需要 350GB 显存（FP16 精度）
- 时间：几天甚至几周
- 费用：数万美元的 GPU 租用费

**现实困境**：
- OpenAI 发布 GPT-3 时，绝大多数公司和研究者根本负担不起全量微调
- 如果给 100 个不同客户微调 100 个版本，就要存储 100 份 350GB 的模型副本

LoRA 的作者（微软研究院团队）在 2021 年提出了一个优雅的解决方案：**我们不直接修改原来的权重，而是在旁边学一个"小补丁"**。

## 1. 什么是矩阵的"秩"？

在讲 LoRA 之前，我们需要理解一个数学概念：**矩阵的秩（Rank）**。

### 1.1 从班级成绩单说起

假设你的班有 30 名同学，参加了 20 门考试，老师整理出一张 30×20 的成绩表：

| 同学 | 数学 | 物理 | 化学 | 英语 | … |
|------|------|------|------|------|---|
| 小明 | 90   | 88   | 85   | 70   | … |
| 小红 | 60   | 58   | 55   | 92   | … |
| …    | …    | …    | …    | …    | … |

这张表有 30×20 = 600 个数字。但仔细一看，你会发现：

> "理科好的同学数学、物理、化学都高；文科好的同学英语、语文都高。"

**整张表的信息，其实可以用少数几个"规律"来概括**。这里只需要两个维度（理科能力 + 文科能力）就能大致还原整张成绩表。

这就是**低秩**的直觉：一个大矩阵，实际上只含有少数几个独立的"信息维度"。

### 1.2 秩的正式定义（面向初中生）

对于一个矩阵：
- **行数** = 有多少行
- **列数** = 有多少列
- **秩** = 这个矩阵中"真正独立的信息"有多少维

**关键性质**：
```
秩(A) ≤ min(行数, 列数)
```

举例：
- 一个 100×100 的矩阵，最大秩 = 100（满秩）
- 如果这个矩阵的秩只有 4，说明它的 100 行可以用 4 个"基础行"的组合来表示

### 1.3 低秩矩阵 = 两个小矩阵的乘积

**重要公式**：任何秩为 r 的矩阵 ΔW（d×k），都可以写成：

```
ΔW = B × A
```

其中：
- B 的形状：d×r（d 行，r 列）
- A 的形状：r×k（r 行，k 列）
- r 远小于 d 和 k（比如 r=4，d=k=1000）

**参数数量对比**：
- 原来的 ΔW：d × k = 1000 × 1000 = **100 万**个参数
- 用 B 和 A 代替：d×r + r×k = 1000×4 + 4×1000 = **8000** 个参数

压缩比：100万 ÷ 8000 = **125 倍**！

## 2. LoRA 的核心想法

### 2.1 预训练权重 + 低秩补丁

Transformer 模型的每一层都有权重矩阵 W（比如 Query/Key/Value 矩阵）。

**全量微调**：直接修改 W
```
W_new = W + ΔW    （ΔW 和 W 一样大）
```

**LoRA**：不动 W，在旁边加一个低秩补丁
```
W_eff = W + B @ A    （B 和 A 加起来比 ΔW 小得多）
```

其中：
- **W**：冻结（freeze），不参与训练，保持预训练参数不变
- **B**：可训练，初始化为**全零矩阵**
- **A**：可训练，初始化为**随机高斯分布**（小数值）

### 2.2 为什么 B 初始化为零？

这是一个精妙的设计！

训练开始时，B=0，所以：
```
B @ A = 零矩阵
W_eff = W + 0 = W
```

这意味着：**LoRA 模型在训练开始时，行为和原始预训练模型完全相同。**

好处：
1. 训练稳定——从原来的"好基线"开始，而不是从随机状态开始
2. 可以直接继承预训练模型的能力

如果 B 也随机初始化，开始时 B@A 就是随机噪声，会破坏原来的能力。

### 2.3 缩放因子 α

实际使用中，LoRA 还加了一个缩放因子：

```
W_eff = W + (α/r) × B @ A
```

- α（alpha）是一个超参数，通常设为 rank 的值（比如 r=4，α=4）
- α/r 起到调节学习率的作用
- 这样换不同的 r 时，不需要重新调整学习率

## 3. 参数量计算对比

### 3.1 GPT-3 的一层 Attention 矩阵

GPT-3 的每个 Attention 层有 4 个投影矩阵：Q、K、V、O，每个形状约为 12288×12288。

| 方法 | 每个矩阵参数量 | 4个矩阵总计 |
|------|--------------|------------|
| **全量微调** | 12288 × 12288 = 1.51 亿 | ~6 亿 |
| **LoRA (r=4)** | 12288×4 + 4×12288 = 98304 | ~39 万 |
| **LoRA (r=8)** | 12288×8 + 8×12288 = 196608 | ~79 万 |
| **LoRA (r=16)** | 12288×16 + 16×12288 = 393216 | ~157 万 |

**LoRA r=4 的压缩比：1.51 亿 ÷ 9.8 万 ≈ 154 倍**

### 3.2 整个 GPT-3 的参数对比

GPT-3 有 96 层，每层有多个矩阵：

| 配置 | 可训练参数 | 相对全量微调 |
|------|----------|------------|
| **全量微调** | 1750 亿 | 100% |
| **LoRA r=4** | 约 470 万 | **0.003%** |
| **LoRA r=8** | 约 940 万 | **0.005%** |

**直觉类比**：
- 全量微调：搬迁整栋房子（175吨）
- LoRA：只换几个灯泡和插座（5公斤）

房子的功能基本保留，但你添加了新的"定制功能"。

### 3.3 存储优势

部署 100 个不同任务的微调版本：

| 方法 | 存储需求 |
|------|---------|
| 全量微调 × 100 | 350 GB × 100 = 35 TB |
| LoRA r=4 × 100 | 350 GB（基础模型）+ 约 10 MB × 100 = **351 GB** |

LoRA 让"一个基础模型 + N 个小适配器"成为可能。

## 5. LoRA 的影响：从论文到生态

### 5.1 论文发布时间线

- **2021-06**：LoRA 论文上传 arXiv（arXiv:2106.09685）
- **2022-05**：在 ICLR 2022 正式发表
- **2023-02**：LLaMA 发布，社区立刻用 LoRA 微调 → Alpaca-LoRA（一周内出现）
- **2023-03**：Stable Diffusion 的 LoRA 微调成为标准工具
- **至今**：HuggingFace PEFT 库、FastChat 等框架将 LoRA 作为默认微调方案

### 5.2 为什么 LoRA 改变了 AI 生态？

**之前（2021年前）**：
- 微调大模型 = 需要大公司级别的 GPU 资源
- 个人研究者、小公司无法参与大模型微调

**之后（LoRA之后）**：
- 一块消费级 GPU（24GB VRAM）可以微调 7B 参数模型
- Stable Diffusion 的个性化风格训练（"训练一个自己的画风"）变得可能
- 开源社区爆炸性增长：数以千计的 LoRA 适配器被分享

### 5.3 LoRA 的后续变体

| 技术 | 年份 | 改进点 |
|------|------|--------|
| **QLoRA** (Dettmers et al.) | 2023 | 结合量化（4-bit），进一步降低显存需求 |
| **AdaLoRA** | 2023 | 自动分配不同层的 rank |
| **LoRA+** | 2024 | 为 A 和 B 使用不同学习率 |
| **DoRA** | 2024 | 分解权重的"方向"和"大小" |
