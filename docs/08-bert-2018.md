# 节点08：BERT（2018）——双向预训练，让 Transformer 真正"理解"上下文

> **所处时代**：2018 年秋。Transformer 已经让机器翻译达到新高度，GPT-1 用 Transformer 做了单向语言模型预训练。但研究者们意识到：一个词的含义，往往取决于它**左边和右边**的上下文——而单向模型永远只能"往后看"。Devlin 等人的问题是：**能不能用 Transformer 做真正的双向预训练？**

---

## 背景故事：词向量的两代困境

### 第一代：上下文无关的词向量（word2vec / GloVe）

2013 年 word2vec 出现后，研究者第一次能把词表示成向量。但这些向量**与上下文无关**：

```
"银行" (河岸) = "银行" (金融机构) = 同一个向量
```

"我去河边的**银行**钓鱼" 和 "我去**银行**存钱" 里，"银行"是完全不同的意思，但 word2vec 给它同一个向量。语言天然有歧义，上下文无关的词向量在需要理解语义的任务上先天受限。

### 第二代：ELMo 和 GPT-1——各解决一半

**ELMo**（2018 年 2 月，Peters 等人）的思路：用双向 LSTM 预训练，每个词的表示随上下文变化。

```
"银行" 在 "河边的银行" 中 → 向量 A（河岸语义）
"银行" 在 "存钱的银行" 中 → 向量 B（金融语义）
```

ELMo 解决了上下文感知的问题，但它用的是 LSTM，表达能力有限，并行训练也慢。

**GPT-1**（2018 年 6 月，Radford 等人）换成了 Transformer，训练速度和表达能力都更强——但它是**单向**的：每个词只能"看到"它左边的词。

```
"它穿过街道，因为 [?] 太累了"
→ GPT-1 只看左边："它穿过街道，因为"，无法利用"太累了"来判断[?]指代谁
```

对于生成任务，单向完全合理（你写作时也不知道后面写什么）。但对于**理解**任务——问答、情感分析、命名实体识别——双向上下文是关键信息，单向白白丢掉了右边的线索。

### BERT 的洞察

> **训练一个双向 Transformer 编码器，让它同时利用左右两侧上下文。**

问题是：如果模型能同时看到左右，那训练"预测下一个词"就太容易了（可以直接抄右边的答案）。BERT 的解决方案是发明了一种新的训练任务——**Masked Language Model（遮罩语言模型）**。

---

## 核心机制一：Masked Language Model（MLM）

### 直觉

期末考试的填空题：**"The cat sat on the ___."**

你不能"抄"答案（答案被遮住了），只能凭上下文推断。BERT 就是用这种方式训练的：随机遮住一些词，让模型根据剩余的词来预测被遮住的词。

### 实现细节

对于输入序列中的每个词：
- **15% 的概率**被选中（称为"目标词"）
- 被选中的词中，再按以下比例处理：
  - **80%** 替换为特殊标记 `[MASK]`
  - **10%** 替换为词汇表中的随机词
  - **10%** 保持原词不变

**为什么不全部换成 `[MASK]`？**

如果全换成 `[MASK]`，模型会学到一个"捷径"：只在看到 `[MASK]` 时才努力预测。但在下游 fine-tuning 任务中，输入里根本没有 `[MASK]`！这种**预训练-推理不一致**会损害性能。

用 10% 随机词 + 10% 原词的目的是：强迫模型对**每一个词的表示**都保持警觉，无论它是否被遮住。

### 数学小补丁：Softmax 分类

MLM 的本质是一个分类任务：对词汇表（通常 3 万个词）做分类，预测哪个词被遮住了。

对于被遮住的位置 $i$，模型输出一个隐向量 $\mathbf{h}_i \in \mathbb{R}^d$，然后通过一个线性层映射到词汇表上：

$$\text{logits}_i = \mathbf{h}_i \mathbf{W}^\top + \mathbf{b} \in \mathbb{R}^{|V|}$$

转化为概率（softmax）：

$$P(\text{词 } j | \text{上下文}) = \frac{e^{\text{logit}_{ij}}}{\sum_{k=1}^{|V|} e^{\text{logit}_{ik}}}$$

损失函数（交叉熵，对每个被遮住的位置求平均）：

$$\mathcal{L}_\text{MLM} = -\frac{1}{|M|} \sum_{i \in M} \log P(\text{原始词}_i | \text{上下文})$$

其中 $M$ 是被遮住的位置集合。

## 核心机制二：Next Sentence Prediction（NSP）

### 动机

许多下游任务需要理解**两个句子的关系**：

- 问答（Question Answering）：问题与答案段落是否相关？
- 自然语言推理（NLI）：前提与假设是矛盾、蕴含还是中立？

为此，BERT 在预训练阶段加入了第二个任务：给定两个句子 A 和 B，判断 B 是否是 A 在原文中的下一句。

### 构造训练数据

- **50%** 正样本：B 确实是 A 的下一句（标签：`IsNext`）
- **50%** 负样本：B 从语料库中随机采样（标签：`NotNext`）

NSP 是一个二分类任务，用 `[CLS]` 位置的输出向量做分类：

$$\text{NSP 损失} = -[\text{label} \cdot \log p + (1-\text{label}) \cdot \log(1-p)]$$

> **后续注记**：RoBERTa（2019）的实验表明 NSP 对多数任务帮助有限甚至有害——原因是 NSP 使用整段话而非单句，模型可能在学习"话题是否相同"而非"是否相邻"。RoBERTa 去掉了 NSP，效果反而更好。这是一个很好的例子：消融实验（ablation study）才是检验设计决策的方法。

---

## 输入表示：三种嵌入的叠加

BERT 的每个输入词向量是三种嵌入之和：

$$\text{输入}_{i} = \text{TokenEmb}(\text{词 }i) + \text{PosEmb}(i) + \text{SegEmb}(\text{句子 A/B})$$

### 格式：[CLS] + 句子 A + [SEP] + 句子 B + [SEP]

```
[CLS]  我  爱  北京  [SEP]  故宫  很  壮观  [SEP]
  0    0   0   0     0      1    1    1     1     ← Segment ID
  0    1   2   3     4      5    6    7     8     ← Position
```

- **Token Embedding**：词汇表中每个词的可学习向量（维度 768）
- **Position Embedding**：与 Transformer 不同，BERT 用的是**可学习的**位置嵌入（而非固定的 sin/cos）。最大支持 512 个位置。
- **Segment Embedding**：句子 A 对应向量 $E_A$，句子 B 对应向量 $E_B$，帮助模型区分哪些词属于哪个句子。

### [CLS] 的特殊作用

`[CLS]`（Classification token）放在最开头。经过所有 Transformer 层后，`[CLS]` 的输出向量会聚合整个序列的语义信息——这个向量被用于分类任务（NSP、句子分类等）。

---

## 架构：Transformer Encoder 叠深

BERT 没有发明新的层，它直接用 Transformer 的 Encoder 部分，叠了很多层：

| 版本 | 层数 L | 隐向量维度 H | 注意力头数 A | 参数量 |
|------|--------|-------------|------------|--------|
| BERT-base | 12 | 768 | 12 | 110M |
| BERT-large | 24 | 1024 | 16 | 340M |

每一层是一个标准的 Transformer Encoder Block：Multi-Head Self-Attention → Add & Norm → FFN → Add & Norm。

因为是双向的（每个位置都能关注所有其他位置），BERT 只用 Encoder，不需要 Decoder。

---

## Fine-tuning：一次预训练，万种任务

BERT 的核心价值主张：**在大规模语料（BookCorpus + Wikipedia，共 3340M 词）上预训练一次，然后在下游任务上 fine-tune**。

Fine-tuning 非常简单：在 `[CLS]` 向量上加一个线性层，然后端到端地用标注数据调整所有参数。

```
句子分类任务：
  输入 → BERT → [CLS]向量 → 线性层(768 → num_classes) → Softmax

问答任务（SQuAD）：
  输入: [CLS] 问题 [SEP] 段落 [SEP]
  输出: 对每个词预测"开始位置"和"结束位置"的概率
```

BERT 论文在 11 个 NLP 任务上刷新了 SOTA，包括 GLUE benchmark（多任务评测）、SQuAD 问答、CoNLL 命名实体识别等。

### 数学小补丁：分类头的形状

设 BERT 最后一层 `[CLS]` 的输出为 $\mathbf{c} \in \mathbb{R}^{H}$（$H=768$），分类头：

$$\text{logits} = \mathbf{c} \mathbf{W}_\text{cls}^\top + \mathbf{b}_\text{cls}, \quad \mathbf{W}_\text{cls} \in \mathbb{R}^{C \times H}$$

其中 $C$ 是类别数。对于情感分析（正面/负面）$C=2$，对于 MNLI 蕴含推理 $C=3$。

## 局限与遗留问题

### 1. 预训练代价极高

BERT-base 在 16 块 TPUv3 上训练 4 天，BERT-large 需要 64 块 TPU 训练 4 天。这在 2018 年意味着只有少数机构能负担得起——这是大模型时代"富者愈富"问题的早期信号。

### 2. MLM 训练效率低

每次只预测 15% 的词，其余 85% 的 token 的损失贡献为零。ELECTRA（2020）用"替换检测"任务解决了这个问题——对每个 token 都预测"它是否被生成器替换过"，训练效率大幅提升。

### 3. 无法直接生成文本

双向注意力意味着每个位置都能看到整个序列，这对理解任务很好，但对生成任务是致命的——生成时你不知道"未来"的内容。GPT 系列保持单向（因果）注意力，专注于生成。

### 4. [MASK] 与推理不一致

如前所述，预训练时有 15% 的 `[MASK]` token，推理时没有，存在分布偏移。SpanBERT、RoBERTa、ALBERT 等后续工作都对此做了改进。

---

## 局限催生下一代突破

| 问题 | 解决方案 |
|------|---------|
| NSP 效果存疑 | **RoBERTa**（2019）：去掉 NSP，动态 masking，更多数据 |
| 参数量太大 | **ALBERT**（2020）：参数共享 + 嵌入因子分解，比 BERT-base 小 18× |
| MLM 效率低 | **ELECTRA**（2020）：替换检测，训练效率 4× |
| 无法生成 | **GPT-2**（2019）：更大的单向 Transformer，few-shot 能力 |
| 双向+生成皆要 | **UniLM**（2019）、**BART**（2019）：混合注意力掩码 |

BERT 的影响远不止 NLP：它展示了"大规模无监督预训练 + 轻量 fine-tuning"这个范式的威力，随后 CV 领域的 ViT、多模态的 CLIP，乃至 GPT-4，都沿着这条路走来。
