# 写作策略

状态：活跃  
建立：2026-04-19（session 20260419-021418）

## 核心约束

本知识库的读者是 14 岁学生，有基础代数和简单 Python，没学过微积分。
**读者看不懂 = 任务失败。**

## 写作顺序（每个知识节点必须遵守）

```
1. 开篇钩子          ← 从读者已知的现象开始（不是从定义）
2. 旧方法的失败      ← 为什么需要这个突破？
3. 直觉/类比         ← 用 14 岁学生的日常生活类比
4. 具体代码          ← 可运行的 Python，从零实现
5. 数学符号（可选）  ← 公式是代码的注解，放最后
6. 局限与引出        ← 这个突破又有什么问题？引出下一个节点
```

禁止顺序：定义 → 公式 → 例子

## 类比来源（面向 14 岁）

✅ 允许：考试分数、足球比赛、游戏分数、去不去操场、辨别手写字/猫
❌ 禁止：股市、绩效考核、企业决策、投资组合

## 大师参照

写完任何节点，用 `refs/masters/writing-for-kids.md` 里的 Rubric 自检。
所有原则都来自真实样本（Karpathy、Nielsen），不是训练记忆均值。

## 自检通过标准

写完一段后问：「如果我把这段念给一个 14 岁学生，
他需要先有什么前置知识才能听懂？」
如果前置知识超出了初中代数+基础 Python，就重写。
