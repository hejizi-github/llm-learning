# RULES — 宪法级约束

这些规则**不可违反**。违反即回滚本次 session。

## 框架自保

1. **禁止修改宪法文件本身** — `.evolve/config.toml`、`.evolve/constitution/` 下的文件 Agent 不得修改（只有用户能改）。
2. **每次 session 只做一件事** — 一次 session 对应一个聚焦的改动（新增一个节点 / 修一类问题 / 建一个工具）。不要"顺手"大规模重构。
3. **所有变更必须 git commit** — session 结束前必须提交，保留可回滚的历史。commit message 要说清这次做了什么。
4. **session 日志写入 journal** — 每次 session 必须在 `.evolve/journal/JOURNAL.md` 留痕：做了什么、KPI 怎么变的、下次该做什么。

## 内容质量门控（硬性）

5. **notebook 必须跑通** — `broken_notebook_ratio` 必须为 0。跑不通的 notebook 不得 commit。
6. **引用必须验证** — 所有引用必须在 `references.bib`，且通过 `cite-verify` 核查。未验证引用不得进入 md。
7. **数学自包含** — 任何超出初中代数的数学概念，第一次出现时必须有面向初中生的讲解（可以在同页，也可以链接到"数学小补丁"子页）。
8. **禁止幻觉引用** — 不确定的论文/作者/年份，要么去查清，要么不要写。宁可空着也不要编。

## 数据源纪律

9. **优先原始来源** — 论文优先 arxiv / 官方出版社。历史事实优先作者自传、官方档案、权威历史书。
10. **禁用社交媒体作为事实依据** — Twitter / Reddit / 知乎/ 小红书等不得作为引用来源。可以用来发现线索，但必须回溯到原始出处。
11. **二手博客需交叉验证** — 如果引用博客/教程，必须在 `citations.jsonl` 里标注"二手"，并至少有一条一手来源佐证。

## 可读性约束

12. **面向初中生** — 每一段都要问自己："一个会基础代数和 Python 的 14 岁读者，读这一段能懂吗？" 不能懂就改。
13. **优先直觉 → 再上数学** — 每个概念先讲直觉/类比/故事，再给形式化定义，再上公式。不要反过来。
14. **代码要手撕** — notebook 里的核心算法必须从零实现一遍（可以之后再对比用库的写法），不要直接 `from sklearn import ...` 就完事。

## 自检与护栏

15. **先建自检，再扩内容** — 第一次迭代必须先把验证脚本建起来（`notebook-run`、`cite-verify`、md 链接检查、可读性 rubric）。没有自检不得大规模扩张。
16. **每次 session 跑一遍自检** — 结束前跑验证脚本，结果写进 journal。任何护栏指标越界，本次改动回滚。
17. **工具放在 `tools/`** — Agent 自建的工具脚本放在仓库的 `tools/` 目录，可被所有 session 复用。

## 用户信号

18. **尊重用户约束** — 用户在 `.evolve/inbox/` 留的指示优先于 Agent 自己的计划。
19. **遇阻求助** — 实在搞不定的问题，写到 `.evolve/help_requests/` 等用户答复，不要猜着硬推。
