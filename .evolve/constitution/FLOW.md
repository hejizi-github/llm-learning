# FLOW — 工作流说明

## 这是什么

一个**自我迭代的教学知识库**：由 Agent 沿着机器学习的发展历史脉络，持续生成并打磨面向初中生的学习内容（md 文档 + 可运行 Jupyter notebook）。

## 核心理念

> **所有事物的复杂度都是逐级递增的。** 找到它演化的脉络，学习门槛就降下来。

所以这个知识库不是按"主题分类"组织，而是按**发展脉络**组织：每一个新概念的出现，都要交代它**为什么出现**（前人的局限）、**它解决了什么**、**它又留下了什么问题**（下一个突破的种子）。

## 目标读者

- 具备**初中数学**基础（会基本代数，可能没学过微积分）
- 会**基础 Python**（能读懂函数、循环、list/dict）
- 好奇心强，想理解"AI 到底是怎么一步步发展出来的"

## 内容形态

每个"知识节点"至少包含：

1. **背景故事** — 这个想法出现的时代背景、前人的困境
2. **原理讲解** — 用初中生能懂的语言解释核心机制
3. **数学自包含** — 涉及的数学概念必须自带讲解，不假设读者懂
4. **可运行 notebook** — 从零手撕关键算法，可复现
5. **局限与衔接** — 这个方法的边界在哪、它如何催生下一个突破
6. **引用溯源** — 所有论文 / 历史事实都必须有可验证的原始链接

## Agent 的自由度

组织形式（时间线 / 主题并行 / 依赖图）、每次迭代的粒度（新增节点 / 打补丁 / 重构）、具体文档风格——**都由 Agent 自主判断**。前提是产出必须**优秀、有深度、让读者真能学到东西**。

## 工作循环

每次 session，Agent 大致按以下步骤推进：

1. **读现状** — 读 memory/active.md，看知识库当前状态和上次的 learnings
2. **定计划** — 今天做什么：扩新节点？修老节点？补引用？建工具？
3. **动手** — 检索（WebSearch / WebFetch / Playwright） → 辨别真伪 → 写作 / 写代码
4. **自检** — 跑验证脚本：notebook 能跑？引用可查？可读性达标？
5. **复盘** — 在 journal/JOURNAL.md 记录本次改动、KPI 变化、新的 learnings
6. **提交** — git commit，保留可回滚的历史

## 反幻觉机制

LLM 容易编造引用、伪造历史细节。为此：

- 所有引用必须落到 `references.bib`，且经 `cite-verify` 工具核查 DOI / arxiv id / 作者 / 年份
- `citations.jsonl` 记录每条引用的溯源（谁第一次引、出现在哪个节点）
- **未通过验证的引用不得进入最终文档**（门控）
- 优先信源：arxiv、作者官方主页、学术出版社、权威历史档案
- 不得使用社交媒体作为事实依据

## 无限迭代

这是一个**没有终点**的工作流。Agent 每次只做可控的一步，但每一步都必须：

- 比上一次**更深**（或更广，或更准）
- 不破坏已有的正确内容
- 留下足够的痕迹，让下一次迭代能无缝接上
