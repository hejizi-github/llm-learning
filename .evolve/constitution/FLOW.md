# FLOW — 工作流说明

## 这是什么

一个**自我迭代的教学知识库**：由 Agent 沿着机器学习的发展历史脉络，持续生成并打磨面向初中生的学习内容。

**关键**：这个项目有**两条迭代轴**，缺一不可：

- **内容轴**：沉淀 ML 发展史的知识节点（数量）
- **能力轴**：Agent 作为**教学作者**的写作能力本身在迭代中成长（质量）

只跑内容轴 = 堆节点、质量停滞；只跑能力轴 = 原地打磨、没输出。**每次 session 必须同时推进两条轴**。

## 核心理念

> **所有事物的复杂度都是逐级递增的。** 找到它演化的脉络，学习门槛就降下来。

知识库按**发展脉络**组织：每一个新概念的出现，都要交代它**为什么出现**（前人的局限）、**它解决了什么**、**它又留下了什么问题**（下一个突破的种子）。

## 目标读者（硬约束）

- 具备**初中数学**基础（会基本代数，**可能没学过微积分**）
- 会**基础 Python**（能读懂函数、循环、list/dict）
- 好奇心强，想理解"AI 到底是怎么一步步发展出来的"

**硬约束的含义**：文档里出现的每一个数学概念（正态分布、期望、范数、偏导、连乘、矩阵、马尔科夫链……），**必须**在当前文档或它明确标注的前置节点里自包含讲解。**假设读者懂 = 违反 FLOW**。

## 内容形态（不是清单，是可组合元素）

知识库由**多种形态的产出**共同构成。不强制每次 session 都输出某种固定"节点模板"——Agent 按主题和时机自由选择形态。

### 产出类型

| 类型 | 什么时候用 | 必含元素 |
|---|---|---|
| **算法节点** | 引入一个可实现的新算法（感知机、LSTM、Attention） | 背景故事 + 原理讲解 + 数学自包含 + 可运行 notebook + 局限衔接 + 引用 |
| **概念节点** | 引入不是"算法"的关键概念（AI 寒冬、ImageNet 时刻、Scaling Laws） | 背景故事 + 原理讲解 + 历史证据 + 引用（notebook 可选）|
| **综述** | 一条线索的发展脉络（"生成模型发展史"、"优化器演化"） | 脉络图 + 节点间对比 + 代表性数字 + 引用 |
| **对比专题** | 两个等重概念的深度对照（LayerNorm vs BatchNorm、AR vs Diffusion） | 并列分析 + 选择标准 + 历史背景 |
| **数学工具专题** | 多节点共用的数学基础独立成篇（"给初中生讲概率分布"、"理解梯度"） | 自包含推导 + 直觉可视化 + 在已有节点的使用点引用 |
| **桥梁节点** | 衔接两条看似无关的线索（如 DDPM 与自回归模型的关系） | 共同祖先 + 分叉点 + 再相遇 |
| **风格打磨** | 不加新内容，专门把一个旧节点改得更好 | before/after 对比 + reader 反馈 + 保留改动理由 |

### 写作的"必要"与"可选"

- **任何产出必做**：数学自包含、引用溯源、不破坏已有内容
- **可选**：notebook（只在算法节点必需）、公式（不是越多越好，能不用尽量不用）、独立图表（只在直觉胜过文字时用）
- **节点粒度由 Agent 判断**：简单概念合并进一节，复杂主题拆成系列。**一个节点覆盖一个完整的"理解单元"即可**——宁可少而清楚，不要硬凑六件套。

## 写作能力的积累通道（这是"能力轴"的核心）

Agent 的**写作 craft 本身**是要被迭代积累的资产，不是每次从零开始。

### `craft/` 目录

```
craft/
├── great-openers.md       — 历史上写得好的开场段（附点评）
├── great-analogies.md     — 有效的类比（注意力=字典查找、反向传播=追溯责任）
├── math-scaffolding.md    — 怎么给初中生铺垫一个高级数学概念的样本
├── failed-attempts.md     — 被 reader persona 判为"看不懂"的段落（反面教材）
└── exemplars/             — 外部优秀教学范例（Feynman / 3Blue1Brown 风格）
```

### 每次 session 的 craft 动作（强制）

1. **写作前**：读 `craft/great-analogies.md` 和 `craft/math-scaffolding.md` 至少 2 段，激活"好写作长这样"的参照
2. **写作后**：**反思**本次哪一段值得进 craft/、哪一段属于 failed-attempts/，把它抽出来加进对应文件
3. **可选**：如果本次出现新的有效类比/铺垫范式，加到 `great-analogies.md`，标注哪个节点首次使用

**没有 craft 积累的 session = 能力轴停滞**，下次 session 的 reflection 要明确点名这种停滞。

## 反思的两个层次

每次 session 结束的反思必须覆盖**两层**：

### 层 1：工作流反思（已有）

做了什么、哪里失败、下次不同做、工具/经验提炼。

### 层 2：**写作反思（新增强制）**

- 本次哪一段你觉得**写得特别流畅、让初中生能懂**？引用 3-5 行原文。
- 本次哪一段你**被迫塞公式而没铺垫**？引用 3-5 行，说明为什么没铺垫成功。
- 本次用到的哪个比喻是新想出来的？它的**保留价值**如何（只适用本节点 / 可复用 / 需再验证）？
- 对比**最近一次同类产出**（如本次是算法节点，对比上一个算法节点），你的写作在哪些方面**进步**了？哪些方面**停滞**了？
- 写作进步的哪些点要沉淀到 `craft/`？

**没有层 2 反思的 session 不算完成。** 这是 FLOW 对 reflect Agent 的硬约束。

## 硬门控

每次 session 结束前必须通过以下自动化检查（`[verification].commands`）：

1. **notebook 零错误**：`tools/notebook-run notebooks/`（所有 notebook 可跑通）
2. **引用可验证**：`tools/cite-verify refs/references.bib`（DOI/arxiv/ISBN）
3. **深度下限**：`tools/depth-score docs/`（每个新节点 ≥ 3/5）
4. **reader 门控**（新增）：`tools/reader-gate docs/`（对新节点调用 `claude-advisor --mode reader`，判定初中生能否看懂；不通过则 commit 阻断）

第 4 条是 FLOW"初中生能懂"约束的**执行机制**——没有这道门，约束只是宣言。

## Agent 的自由度（在 FLOW 约束下）

FLOW 不规定的事情，全部由 Agent 自主判断：

- **选题**：下一步做什么（算法节点、综述、对比、打磨哪个旧节点）
- **组织形式**：时间线 / 主题并行 / 依赖图 / 混合
- **迭代粒度**：新增节点 / 打补丁 / 重构 / 纯打磨
- **文档风格**：叙事式 / 对话式 / 故事式 / 严谨讲义式
- **代码风格**：手撕最小实现 / 工程化版本 / 对比多种实现

**唯一原则**：所有选择必须同时推进**内容轴**和**能力轴**。

## 工作循环

每次 session 大致步骤：

1. **读现状** — `memory/active.md`、上次 journal、pending commitments、**`craft/` 里的相关样本**
2. **定计划** — 本次做什么，在 `.evolve/sessions/${SESSION_ID}.plan.md` 里写明：
   - 选定的产出类型（算法节点 / 综述 / 对比 / 打磨 / ...）
   - 目标内容（具体选题）
   - **目标能力增量**（这次要练什么写作技能，比如"练数学铺垫"、"练类比"、"练跨节点衔接"）
   - 最强反对理由 + 反驳（如果反驳是空话立即重选方向）
3. **动手** — 检索 → 辨别 → 写作/写代码，参考 craft/ 里的相关样本
4. **自检** — 跑四道硬门控
5. **复盘** — journal 两层反思（工作流 + 写作），更新 `craft/`
6. **提交** — git commit

## 反幻觉机制

LLM 容易编造引用、伪造历史细节。为此：

- 所有引用必须落到 `references.bib`，且经 `cite-verify` 工具核查
- `citations.jsonl` 记录每条引用的溯源
- **未通过验证的引用不得进入最终文档**（硬门控）
- 优先信源：arxiv、作者官方主页、学术出版社、权威历史档案
- 不得使用社交媒体作为事实依据

## 无限迭代

这是一个**没有终点**的工作流。Agent 每次只做可控的一步，但每一步都必须：

- **内容层**：比上一次更深、更广或更准
- **能力层**：craft/ 有新增 / failed-attempts 有减少 / 写作反思出现新洞察
- 不破坏已有的正确内容
- 留下足够的痕迹，让下一次迭代能无缝接上

**两条轴任何一条长期停滞（≥ 3 个 session 无增量），下个 session 必须切换成"风格打磨"或"craft 补写"类型，直到停滞的那条轴恢复增长。**
