# FLOW

## 目标

以机器学习发展史为骨架的知识库。

## 读者

14 岁学生：会基础代数，会简单 Python，没学过微积分。

**读者看不懂 = 任务失败。**

## 四条不可违反

1. **引用必须真** — 查不到 DOI / arxiv / ISBN 的不许写
2. **论文必须真 fetch 过** —
   任何"论文 X 说了 Y"的陈述必须基于真下载到磁盘的原文。
   凭训练记忆复述 = 幻觉 = 失败。
3. **Notebook 必须能跑** — `jupyter nbconvert --execute` 零错误
4. **读者看不懂的文档不算文档** — 出现初中生不懂的数学概念时，
   必须在当前文档或它明确链接的前置文档里自包含讲解

## 深度优先于速度

**一个 session 不必做完一切，可以分多个 session 做一件事。**

- 引用论文 → 先用一个 session **深度学习**这篇论文：
  写 `refs/notes/{id}.md`（key claims / 关键数字 / 你的疑问 / 论文局限）
  下一个 session 才能基于它写文档。

- 写文章 → 先用一个 session **深度思考怎么写**：
  这个主题的受众理解门槛在哪、好类比是什么、结构顺序如何
  写 `craft/{topic}-plan.md`
  下一个 session 才动手写。

- 看代码 → 先用一个 session **深度阅读**：
  画架构图、找关键路径、记不懂的问题到 `refs/code-notes/`
  下一个 session 才做改动。

- **不允许一次 session 做完 "读论文 + 想方法 + 写文档 + 写 notebook + 写测试"**。
  这是快速 pattern matching，不是深度思考。

每个 session 的产出可以是**纯 notes / plan / draft**，也算合格产出——
只要它是真深度思考的痕迹，不是脑内 greedy 拼凑。

如果一个 session 的产出是"10 分钟秒生成的三件套"——违反此原则。
下次 session 必须补做深度部分。

## 循环的意义：累积真经验

每次 session 必须产出**可复用的资产**，供未来 session 调用。
资产不是你自说自话的反思，而是**真从外部世界拿回来的东西**：
真读过的论文摘要、有效的教学类比、被验证过的方法模式。

如果第 N 次 session 和第 1 次 session 能用的资产一样多——循环没产生价值。

资产的存放位置、格式、分类方式由你决定。

## 其他一切

**由你决定。**

- 什么时候做什么、做什么、用什么形态由你选
- 评估维度除上面四条之外由你自己定和演化
- 流程、反思结构、工具、资产目录结构都由你

**你比我聪明，规则少 = 你的空间大。**

## 规则的演化

如果你发现某条规则反复撞墙，提案加上（写到 `.evolve/proposals/`）。  
如果你发现某条规则没必要，提案删除。

FLOW 本身也是可以被你修改的——只是要走提案和审批。
